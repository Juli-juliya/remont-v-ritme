# Ремонт в ритме

Мобильный прототип Telegram Mini App для контроля нескольких ремонтов: общий прогресс, отдельные бюджеты и сроки, работы по комнатам и закупка материалов.

Интерфейс оформлен в светлой уютной системе: сливочный фон, песочные и овсяные поверхности, карамельные действия, мягкие тени и спокойный контраст натуральных материалов.

## Что уже работает

- несколько независимых объектов с быстрым переключением;
- отдельные бюджет, смета и дедлайн каждого объекта;
- добавление и удаление пространств с площадями;
- количество розеток и выключателей в каждой комнате;
- загрузка изображения планировки;
- пакеты работ, привязанные сразу к нескольким комнатам;
- даты начала и окончания, стоимость и прогресс каждой работы;
- фильтры «Все», «В работе», «Риски»;
- материалы, предметы, мебель и расходники для одной или нескольких комнат;
- раздельные показатели «в смете» и «фактически оплачено»;
- быстрое добавление работы, материала или расхода;
- сохранение данных в `localStorage` и миграция состояния раннего прототипа;
- адаптация под Telegram WebApp API, safe-area и тактильный отклик.

## Логика прогресса

Итоговый процент — не просто доля закрытых галочек. В MVP он учитывает:

- 75% — фактическое выполнение работ;
- 15% — здоровье бюджета;
- 10% — соблюдение общего срока.

Для production-версии срок лучше считать по критическому пути, а бюджет — по связке `план / обязательства / оплачено`.

## Локальный запуск

```bash
npm install
npm run dev
```

Приложение откроется по адресу `http://localhost:5173`.

## Production-сборка

```bash
npm run build
```

Статические файлы появятся в `dist/`. Их можно разместить на Vercel, Netlify или другом HTTPS-хостинге и указать URL как Main Mini App в BotFather.

## Следующий технический слой

Для командной и многопользовательской версии понадобятся PostgreSQL/Supabase, серверная проверка Telegram `initData`, роли владельца/подрядчика, загрузка чеков и напоминания через бота.
